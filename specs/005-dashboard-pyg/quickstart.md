# Quickstart: Sprint 05 - Dashboard PYG y Posición

This quickstart describes how to validate the sprint once implemented. Exact commands will be added when tooling exists.

## Preconditions

- Current sprint spec has an approved plan.
- Test tooling exists or is created by this sprint.
- Required prior sprints are complete.

## Validation Path

1. Validate story: Ver resultado anual (panel PYG).
2. Validate story: Ver evolución mensual (panel PYG).
3. Validate story: Ver Top 10 ingresos y egresos (panel PYG).
4. Validate story: Ver qué tengo y qué debo (panel de posición).

## Expected Result

The sprint delivers one independently demonstrable dashboard with separate PYG and position panels. PYG totals must not change when only balance saldos change, and vice versa. Out-of-scope formal CxC/CxP document modules are not required.
